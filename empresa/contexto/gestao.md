# KAB — princípios práticos de gestão para líderes

> Uso pretendido: base compartilhável para o Tony orientar gestores da KAB em situações operacionais. Conteúdo sanitizado, sem ata privada nem detalhes sensíveis.

## Princípio central

Gestor não deve levar apenas o problema para cima. Deve levar:

1. **Diagnóstico** — o que está acontecendo, com fatos.
2. **Impacto** — cliente, prazo, qualidade, custo e risco.
3. **Alternativas** — pelo menos 2 caminhos possíveis quando houver tempo.
4. **Recomendação** — qual opção o gestor sugere e por quê.
5. **Próxima ação** — quem faz o quê, até quando.

Se o gestor só leva o impasse, ele transfere a gestão. Se leva opções e recomendação, ele pratica gestão. Sim, é essa a diferença entre “avisar problema” e “liderar”.

## Regra Tony para coaching de gestores

Quando um gestor trouxer um problema operacional, o Tony deve responder no padrão:

- **Entendi o problema. Qual é o impacto?**
- **Quais opções você já considerou?**
- **Qual opção você recomenda?**
- **O que você precisa aprovar/escalar?**
- **Qual é o plano nas próximas horas?**

Tony pode ajudar a estruturar o raciocínio, mas não deve virar atalho para pular responsabilidade gerencial.

## Checklist rápido — problema crítico

Antes de escalar para diretoria/presidência, o gestor deve preparar:

- **Situação atual:** o que ocorreu e desde quando.
- **Cliente/processo afetado:** quem será impactado.
- **Prazo crítico:** data/hora limite.
- **Risco:** qualidade, entrega, segurança, custo ou imagem.
- **Contenção imediata:** o que dá para fazer hoje.
- **Alternativa A:** custo/risco/prazo.
- **Alternativa B:** custo/risco/prazo.
- **Recomendação:** decisão sugerida.
- **Apoio necessário:** quem precisa ajudar ou aprovar.

## Caso de aprendizagem — peça crítica com medição externa

Contexto sanitizado:

- Projeto novo com característica dimensional crítica.
- A empresa ainda não tinha medição tridimensional interna disponível.
- Método alternativo de medição não era confiável para a característica crítica.
- Havia prazo curto de primeira entrega ao cliente.
- A produção precisou operar com contenção, medição externa e ajuste por amostragem.

Lições:

- Para projeto crítico, **metrologia é gate de industrialização**, não detalhe de fim de processo.
- Fornecedor externo de medição deve ser mapeado antes da primeira produção, especialmente quando a medição interna ainda não existe.
- A contenção pode salvar uma entrega, mas não substitui processo robusto.
- Primeira entrega de cliente novo exige plano de contingência antes do lote, não improviso durante o feriado.
- O gestor responsável deve escalar com opções: esperar, produzir parcial, medir externo, alterar processo, comunicar cliente — e recomendar um caminho.

## Frase-padrão que Tony pode usar

> “Antes de eu te ajudar a escalar isso, vamos organizar como gestor: qual é o diagnóstico, quais são as opções, qual você recomenda e qual decisão precisa de aprovação?”
